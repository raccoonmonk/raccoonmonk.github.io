Porting a C++ Qt project to macOS: Requirements

SideBlocks::blog
2020-06-15
distribution, macos

Here are the requirements for porting a C++ Qt Application to macOS. This list will help you to roughly estimate amount of time and costs for the port task.

Machine running macOS

Get a macOS. It can be possibly virtual. Here’s a popular project that looks really promising https://github.com/kholia/OSX-KVM. Again, having a physical Apple machine is not mendatory. But of course it is recommended by Apple. At least license-wise.

Xcode with command line tools

Install Xcode. You can find all versions of Xcode here https://stackoverflow.com/a/10335943. Those are just a collection of links. The actual files are hosted on Apple servers. To download them you will need to log into your Apple Developer Account.

macOS and Qt versions

Make sure that your macOS version and Xcode version are supported by Qt https://doc.qt.io/qt-5/supported-platforms.html. You can start porting with building your project on the initial platform but with Clang and up-to-date Qt version.

Qt libraries

Get Qt libraries. There is no need to build Qt by yourself. It takes a lot of time and can be challenging. Obtaining Qt through online installer is absolutely acceptable https://download.qt.io/archive/online_installers/. It is good on Linux and Windows too. It is easy to update in future and helps to avoid conflicts in system. Even better now since Qt became relocatable https://www.qt.io/blog/qt-is-relocatable.

Get an Apple Developer Account

Optional. You don’t have to have Apple products to be an Apple Developer. Just go and register https://developer.apple.com/account/

Package Manager

Install a package manager. It optional but highly recommended. Likely your project has dependencies or you will need some utilities to configure and/or build it like cmake bison python.
The most popular being Brew https://brew.sh/ and the other one being http://www.finkproject.org/. Again less building more re-using.
